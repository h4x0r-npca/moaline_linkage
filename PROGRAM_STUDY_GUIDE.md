# 모아라인 연동 자동화 프로그램 개발 정리

이 문서는 `모아라인연동자동화.exe`가 어떤 구조로 만들어졌는지, 어떤 라이브러리를 사용했는지, 각 파일이 어떤 역할을 하는지 공부하기 위한 자료입니다.

## 1. 프로그램 목적

이 프로그램은 모아라인 플러스 또는 모아라인 상점 프로그램을 감지한 뒤, 영수증/주문 연동에 필요한 가상 COM 포트를 자동으로 만들고 모아라인 설정 파일을 수정하는 도구입니다.

기존 매크로 방식은 화면 위치, 실행 타이밍, 권한, 창 상태에 따라 실패할 수 있습니다. 이 프로그램은 화면을 클릭하는 방식이 아니라 Windows 레지스트리, 설정 파일, 프로세스, com0com 명령어를 직접 다루기 때문에 더 안정적입니다.

## 2. 사용 기술과 라이브러리

### Python 3.8

실제 배포 exe는 Python 3.8 32-bit 기준으로 빌드했습니다. 모아라인/Windows 환경과 PyInstaller 호환성을 고려해 최신 Python 기능보다는 3.8에서 안정적으로 동작하는 문법과 패키지 버전을 사용했습니다.

### CustomTkinter

UI는 `customtkinter==5.2.2`로 만들었습니다.

기본 `tkinter`보다 버튼, 프레임, 체크박스, 테마 색상 표현이 깔끔해서 데스크톱 유틸리티 UI를 만들기 좋습니다. 이 프로그램에서는 다음에 사용됩니다.

- 메인 창
- 브랜드 헤더
- 진행 단계 패널
- STATUS 표시 영역
- `연동 시작`, `연동 삭제` 버튼
- `낮은연동포트번호 사용` 체크박스
- 완료/오류 팝업

### tkinter

CustomTkinter도 내부적으로 tkinter 위에서 동작합니다. 이 프로그램에서는 로그 출력 영역에 `tkinter.scrolledtext.ScrolledText`를 사용했습니다. 긴 작업 로그를 계속 누적해서 보여주기 편하기 때문입니다.

### Pillow

`Pillow==10.4.0`을 사용해 `webp` 로고 이미지를 읽고, 창 아이콘/브랜드 이미지로 표시합니다.

주요 사용처:

- `로고한글.webp`를 상단 헤더에 표시
- `로고.webp` 또는 `moaline.ico`를 프로그램 아이콘으로 사용

### psutil

`psutil==5.9.5`를 사용해 실행 중인 모아라인 프로세스를 찾아 종료합니다.

연동 설정을 수정한 뒤 모아라인 프로그램을 재시작해야 설정이 적용되므로, 프로세스 종료/재실행에 사용합니다.

### PyInstaller

`pyinstaller==4.5.1`로 Python 코드를 Windows 단일 exe로 패키징합니다.

빌드 옵션:

- `--onefile`: 하나의 exe로 생성
- `--noconsole`: 콘솔창 없이 GUI 앱으로 실행
- `--uac-admin`: 관리자 권한 요청
- `--icon moaline.ico`: exe 아이콘 지정
- `--collect-data customtkinter`: CustomTkinter 리소스 포함
- `--add-data`: 로고 이미지와 아이콘 파일 포함

## 3. 전체 실행 흐름

프로그램 실행 흐름은 크게 네 가지입니다.

1. 관리자 권한 확인
2. UI 생성 및 현재 STATUS 확인
3. 연동 시작 또는 연동 삭제
4. 작업 종료 후 STATUS 재확인

### 실행 직후

`main.py`의 `main()` 함수가 시작점입니다.

```python
def main() -> None:
    admin.ensure_admin()
    root = ctk.CTk()
    App(root)
    root.mainloop()
```

먼저 `admin.ensure_admin()`으로 관리자 권한을 확인합니다. 가상 COM 포트 생성, 레지스트리 조회, 설치 파일 실행 등은 관리자 권한이 필요할 수 있기 때문입니다.

그 다음 CustomTkinter 창을 만들고 `App` 클래스를 실행합니다.

## 4. 연동 시작 흐름

사용자가 `연동 시작` 버튼을 누르면 다음 순서로 동작합니다.

1. 버튼 비활성화
2. 진행 단계 UI 초기화
3. com0com 설치 여부 확인
4. 가상 COM 포트 생성
5. 모아라인 플러스/상점 설치 감지
6. 모아라인 INI 설정 수정
7. 필요한 폴더 생성
8. 모아라인 프로그램 재시작
9. 완료 팝업 표시
10. STATUS 재확인

작업은 별도 스레드에서 실행됩니다. GUI 프로그램에서 긴 작업을 메인 스레드에서 돌리면 화면이 멈추기 때문입니다.

## 5. 연동 삭제 흐름

사용자가 `연동 삭제` 버튼을 누르면 다음 순서로 동작합니다.

1. 모아라인 설치 감지
2. 현재 INI에서 `POS1`, `MOA1` 값 확인
3. 설정 파일 백업 복원 또는 연동 키 제거
4. 해당 가상 COM 포트 페어 제거
5. 모아라인 프로그램 재시작
6. 완료 팝업 표시
7. STATUS 재확인

초기 버전은 기본 포트인 `COM16`, `COM15`만 제거했지만, 지금은 INI에 실제로 들어있는 `POS1`, `MOA1` 기준으로 제거합니다. 낮은 포트 옵션을 써서 `COM1`, `COM2`로 만들었을 때도 삭제가 맞게 동작하기 위한 구조입니다.

## 6. STATUS 판단 기준

프로그램 하단에는 현재 연동 상태가 표시됩니다.

예시:

```text
STATUS : 연동 완료 : (COM16)
STATUS : 연동 미설치
```

연동 완료로 판단하려면 다음 조건을 만족해야 합니다.

### 공통 조건

- 모아라인 플러스 또는 모아라인 상점이 설치되어 있어야 함
- INI 파일에 `POS1`, `MOA1`이 있어야 함
- `POS1`, `MOA1` 값이 `COM숫자` 형식이어야 함
- com0com 실제 포트 목록에 `POS1/MOA1` 쌍이 존재해야 함

### 모아라인 플러스 조건

플러스는 `PRNYN=1`이면 프린터 없음 설정으로 볼 수 있습니다. 따라서 `PRINT1=COM99`가 없어도 연동 완료로 판단할 수 있습니다.

현재 플러스 설정 방식:

```ini
[linkage]
PRNYN=1
POS1=COM16
MOA1=COM15
BaudRate1=9600
```

### 모아라인 상점 조건

상점은 기존 방식대로 `PRINT1`이 `COM숫자` 형식이어야 하고, `linkCOM숫자` 섹션의 `Settings` 값이 baud 설정을 가지고 있어야 합니다.

예시:

```ini
[linkage]
PRINT1=COM99
POS1=COM16
MOA1=COM15

[linkCOM99]
Settings=9600,n,8,1
```

## 7. 낮은연동포트번호 사용 옵션

일부 포스 프로그램은 `COM16`, `COM15` 같은 높은 포트를 선택할 수 없거나, 특정 범위 포트만 설정 가능한 경우가 있습니다.

이를 위해 STATUS 옆에 `낮은연동포트번호 사용` 체크박스를 추가했습니다.

체크하지 않으면 기본 흐름:

- 우선 `COM16`, `COM15` 사용
- 이미 사용 중이면 `COM20~COM40` 범위에서 빈 쌍 탐색

체크하면 낮은 포트 흐름:

- `COM1`부터 순서대로 검색
- 사용 중이지 않은 포트 2개를 찾음
- 찾은 두 포트로 가상 포트 페어 생성

예시:

```text
COM1, COM2가 비어 있으면 COM1 / COM2 페어 생성
COM1이 사용 중이면 COM2 / COM3 또는 다음 빈 포트 조합 생성
```

## 8. 파일별 역할

### main.py

프로그램의 중심 파일입니다.

주요 역할:

- GUI 구성
- 버튼 이벤트 처리
- 진행 단계 표시
- 로그 출력
- STATUS 표시
- 설치/삭제 작업 흐름 제어
- 작업 종료 후 STATUS 갱신
- 창 제목, 로고, 아이콘 설정

핵심 클래스는 `App`입니다.

`App` 안에서 UI를 만들고, `연동 시작`, `연동 삭제`, `STATUS 확인` 작업을 연결합니다.

### admin.py

관리자 권한 확인 담당 파일입니다.

주요 함수:

- `is_admin()`: 현재 프로세스가 관리자 권한인지 확인
- `relaunch_as_admin()`: 관리자 권한으로 프로그램 재실행
- `ensure_admin()`: 권한 없으면 재실행 요청

가상 COM 포트 생성과 드라이버 설치는 관리자 권한이 필요할 수 있으므로 프로그램 시작 시 가장 먼저 실행됩니다.

### constants.py

프로그램 전체에서 사용하는 상수 모음입니다.

포함 내용:

- com0com 다운로드 URL
- com0com 설치 파일명
- com0com 임시 다운로드/압축해제 경로
- 기본 포트 값
- 모아라인 플러스 기본 설치 경로
- 모아라인 상점 기본 설치 경로
- INI 파일명
- 고객센터 URL

상수 파일을 따로 둔 이유는 경로, 포트, URL 같은 값이 여러 파일에서 반복 사용되기 때문입니다.

### com0com_handler.py

com0com 관련 기능을 담당합니다.

주요 역할:

- com0com 설치 여부 확인
- com0com 다운로드
- 압축 해제
- 드라이버 설치
- 사용 중인 COM 포트 조회
- 사용 가능한 포트 페어 찾기
- 낮은 포트 번호 우선 탐색
- 가상 COM 포트 생성
- 가상 COM 포트 목록 파싱
- 가상 COM 포트 제거

중요한 점은 `setupc.exe`를 직접 호출한다는 것입니다. 화면을 클릭하지 않고 com0com의 명령행 도구를 사용하므로 자동화 안정성이 높습니다.

### config_writer.py

모아라인 설정 파일을 읽고 쓰는 파일입니다.

주요 역할:

- INI 파일 백업
- 모아라인 플러스 `LinkAge.ini` 수정
- 모아라인 상점 `Call Star.ini` 수정
- 현재 연동 설정 읽기
- 백업 복원
- 백업이 없을 때 연동 키 제거
- 모아라인 플러스 COM 로그 폴더 생성

플러스는 현재 `PRNYN=1`을 사용해 프린터 없음 방식으로 설정합니다.

상점은 `PRINT1`, `POS1`, `MOA1`, `linkCOM숫자/Settings` 구조를 사용합니다.

### moaline_detector.py

모아라인 프로그램 설치 여부를 찾는 파일입니다.

주요 방식:

- Windows 레지스트리의 Uninstall 목록 조회
- DisplayName에서 모아라인 플러스/상점 이름 확인
- InstallLocation 또는 기본 경로 확인
- exe 파일 존재 여부 확인
- Plus를 우선순위로 정렬

반환 데이터는 `MoalineInstall` dataclass입니다.

```python
kind        # plus 또는 store
display_name
install_dir
exe_path
ini_path
```

### process_handler.py

모아라인 프로그램 재시작 담당 파일입니다.

주요 역할:

- 실행 중인 모아라인 프로세스 종료
- exe 경로로 프로그램 재실행
- 설정 적용을 위해 종료 후 1.5초 대기

프로세스 탐지와 종료에는 `psutil`을 사용합니다.

### build_info.py

버전과 Release 날짜를 저장하는 파일입니다.

현재 값:

```python
APP_VERSION = "1.0"
RELEASE_DATE = "2026-06-01"
```

`main.py`는 이 값을 읽어서 제목 표시줄에 표시합니다.

예시:

```text
모아라인 연동 자동화 Ver 1.0 (Release : 2026-06-01)
```

### build_helper.py

빌드를 자동화하는 파일입니다.

주요 역할:

- 필요한 패키지 설치
- `build_info.py`의 Release 날짜 갱신
- PyInstaller 실행
- 로고, 아이콘, CustomTkinter 리소스 포함
- exe 생성 후 임시 파일명 정리

개발자가 직접 PyInstaller 옵션을 외우지 않아도 `build.bat` 또는 Python 실행으로 빌드할 수 있게 만든 파일입니다.

### build.bat

Windows에서 빌드를 쉽게 실행하기 위한 배치 파일입니다.

역할:

- Python 3.8 32-bit 실행 파일 경로 확인
- `build_helper.py` 실행
- 빌드 완료 후 `dist` 폴더 안내

### requirements.txt

외부 패키지 버전을 기록한 파일입니다.

현재 패키지:

```text
psutil==5.9.5
customtkinter==5.2.2
Pillow==10.4.0
```

### 모아라인연동자동화.spec

PyInstaller가 생성/사용하는 빌드 설정 파일입니다.

주요 내용:

- `main.py`를 시작점으로 설정
- 로고/아이콘 파일 포함
- CustomTkinter 데이터 포함
- 관리자 권한 요청
- exe 아이콘 설정

주의할 점은 `build_helper.py`가 빌드할 때 이 파일을 다시 생성할 수 있다는 점입니다. 보통은 직접 수정하기보다 `build_helper.py`의 옵션을 수정하는 편이 관리하기 쉽습니다.

### 이미지 파일

프로그램 디자인과 아이콘에 사용됩니다.

- `로고.webp`: 모아라인 심볼 로고
- `로고한글.webp`: 상단 헤더의 한글 로고
- `로고.svg`: 원본 벡터 로고
- `moaline.ico`: exe/창 아이콘

## 9. 왜 매크로보다 안정적인가

이 프로그램은 UI 자동 클릭 방식이 아닙니다.

매크로 방식은 다음 문제에 취약합니다.

- 창 위치가 바뀜
- 버튼 위치가 바뀜
- 화면 해상도가 다름
- UAC 권한 창 때문에 흐름이 끊김
- 프로그램 응답 속도가 느리면 클릭 타이밍이 어긋남
- 포트 생성 결과를 정확히 검증하기 어려움

반면 이 프로그램은 다음 방식으로 동작합니다.

- 레지스트리에서 설치 경로 확인
- `setupc.exe` 명령어로 포트 생성/삭제
- INI 파일을 직접 읽고 수정
- 프로세스를 직접 종료/재실행
- 실제 포트 쌍과 INI 설정을 비교해 STATUS 판단

즉, 화면이 아니라 시스템 상태를 직접 조작하고 검증합니다.

## 10. 핵심 설계 포인트

### 작업 흐름과 UI를 분리

UI는 `main.py`에 있고, 실제 기능은 아래 파일들로 분리되어 있습니다.

- com0com 작업: `com0com_handler.py`
- 설정 파일 작업: `config_writer.py`
- 설치 감지: `moaline_detector.py`
- 프로세스 재시작: `process_handler.py`

이 구조 덕분에 특정 기능만 수정하기 쉽습니다.

### 긴 작업은 스레드로 실행

드라이버 설치, 포트 생성, 다운로드, 재시작은 시간이 걸릴 수 있습니다. 이 작업을 GUI 메인 스레드에서 실행하면 화면이 멈춥니다.

그래서 `threading.Thread(..., daemon=True)`로 백그라운드에서 처리하고, UI 업데이트는 `root.after()`로 메인 스레드에 예약합니다.

### 상태 검증을 별도로 둠

STATUS는 단순히 “방금 설치 버튼을 눌렀다”로 판단하지 않습니다.

실제 기준:

- 모아라인 설치 확인
- INI 설정 확인
- com0com 포트 페어 확인

이 세 가지를 다시 읽어서 판단합니다. 그래서 프로그램 재실행 후에도 현재 상태를 알 수 있습니다.

## 11. 개발팀에 설명할 때 짧은 요약

아래처럼 설명하면 됩니다.

```text
Python 3.8 기반 Windows GUI 프로그램입니다.
UI는 CustomTkinter로 만들었고, exe 패키징은 PyInstaller를 사용했습니다.

매크로처럼 화면을 클릭하는 방식이 아니라,
레지스트리에서 모아라인 설치 경로를 찾고,
com0com의 setupc.exe 명령어로 가상 COM 포트를 직접 생성/삭제하며,
모아라인 INI 파일을 configparser로 직접 수정합니다.

설치 후에는 모아라인 프로세스를 psutil로 종료/재시작해서 설정을 반영합니다.
현재 연동 상태도 INI 값과 실제 com0com 포트 쌍을 비교해서 판단합니다.
```

## 12. 유지보수 시 자주 볼 파일

기능 수정이 필요할 때는 보통 아래 파일을 보면 됩니다.

- UI 문구/버튼/화면 수정: `main.py`
- 포트 생성/삭제 방식 수정: `com0com_handler.py`
- INI 설정 방식 수정: `config_writer.py`
- 모아라인 설치 경로 감지 수정: `moaline_detector.py`
- 빌드 옵션/버전 날짜 수정: `build_helper.py`, `build_info.py`

## 13. 빌드 방법

Python 3.8 32-bit가 설치되어 있다는 전제에서 아래 파일을 실행합니다.

```bat
build.bat
```

또는 직접 실행:

```powershell
%LOCALAPPDATA%\Programs\Python\Python38-32\python.exe build_helper.py
```

빌드 결과는 아래 경로에 생성됩니다.

```text
dist\모아라인연동자동화.exe
```
